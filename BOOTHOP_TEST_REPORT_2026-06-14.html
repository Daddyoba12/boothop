Section 5 — All Fixes Applied

13 fixes committed across 3 commits. Listed in commit order.

Commit 3ccc27f — Core flow fixes (cron, barcode, messaging)

1
Messages inbox — complete rewrite
Critical

Problem: The page queried delivery_matches and messages tables from a scrapped pre-P2P schema. These tables do not exist. The inbox showed nothing.

Fix: Full rewrite. Now fetches /api/auth/me for user email, /api/dashboard for matches, filters to MESSAGING_STATUSES = ['active', 'delivery_confirmed', 'disputed'], renders conversation list linking to /messages/[matchId].

src/app/messages/page.tsx
2
Africa-outbound matches — 7-day expiry added to cron
Critical

Problem: awaiting_authorisation status was never included in the expire-matches cron. Africa-outbound trips were locked at pending forever if admin never acted.

Fix: Added a 7-day cutoff bucket cutoff7d querying matches with awaiting_authorisation older than 168 hours. Included in toCancel array with custom cancellation_reason: "Africa-outbound match expired — not authorised within 7 days."

src/app/api/cron/expire-matches/route.ts
3
Cron schedule changed — daily → every 6 hours
High

Problem: 0 3 * * * — ran once per day at 03:00. A 4-hour timeout could wait up to 23 hours before being processed. Match participants stuck in limbo.

Fix: Changed to 0 */6 * * * — runs at 00:00, 06:00, 12:00, 18:00. Max wait for any threshold is now 6 hours.

vercel.json → crons[expire-matches].schedule
4
Barcode scan — match status lifecycle guard
High

Problem: No match status check in scan-sender-barcode. Senders could request traveller location before delivery was active (during matched, agreed, kyc_pending).

Fix: Added guard — only processes scan if match.status is active or delivery_confirmed. Returns { error: 'Location tracking is only available once your delivery is active.', status: 'not_active' } otherwise.

src/app/api/tracking/scan-sender-barcode/route.ts
5
Barcode scan — null barcode guard
High

Problem: If traveller_barcode was null (not yet generated), the response included a tracking URL with /track/null — breaking push notifications and emails.

Fix: Added null check on match.traveller_barcode — returns 400 with message "Traveller barcode not yet assigned for this delivery."

src/app/api/tracking/scan-sender-barcode/route.ts
